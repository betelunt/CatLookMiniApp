# 猫咪广场筛选栏 — 设计文档

> 日期：2026-06-08
> 状态：设计完成，待实现

## 概述

在「猫咪广场」(`pages/index`) 瀑布流上方新增筛选栏：城市选择器 + 品种选择器 + 名字搜索框，三个条件联动过滤取交集。

## 布局

单排三列 pill 风格，与领养页城市筛选保持一致：

```
┌─────────────────────────────────────────┐
│  城市 ▾    │  品种 ▾    │  🔍 搜索      │
└─────────────────────────────────────────┘
```

- 前两个是 `<picker>` 组件，显示当前选中值或默认 "城市" / "品种"
- 第三个是 `<input>` 搜索框，placeholder "🔍 搜索"
- 选中筛选项后 pill 变色激活

## 交互行为

| 条件 | 触发方式 | 行为 |
|------|---------|------|
| 城市 | Picker 省市区三级联动 | 选中即时过滤 |
| 品种 | Picker selector | 选中即时过滤 |
| 名字 | Input 输入 | 300ms 防抖实时搜索 |
| 三者关系 | 联动取交集 | 任一变化 → 重新过滤 |

### 激活指示

当有筛选项选中时，筛选栏下方显示激活标签行：

```
[北京 ✕] [橘猫 ✕]  3 只猫咪
```

点击 `✕` 清除对应筛选条件。

### 结果计数

顶部标题同步更新过滤后数量：`全部猫咪 (3)`

### 无结果时

保持瀑布流空状态，显示「没有找到匹配的猫咪」。

## 涉及文件

仅 `pages/index/` 三个文件：

| 文件 | 改动 |
|------|------|
| `index.wxml` | 新增筛选栏 + 激活标签行 |
| `index.wxss` | pill 按钮样式 + 搜索框 + 激活标签 |
| `index.js` | 城市/品种 picker 数据 + 搜索防抖 + 三条件 filter |

## 数据来源

- 城市列表：从全部猫咪数据中动态提取（同领养页逻辑）
- 品种列表：复用 `BREEDS` 常量
- 搜索：匹配 `name` 字段

## 与现有功能兼容

- 瀑布流双列布局不变
- 骨架屏、空状态保持不变
- 下拉刷新后保留筛选条件
- DEV_MODE 切换逻辑不变
